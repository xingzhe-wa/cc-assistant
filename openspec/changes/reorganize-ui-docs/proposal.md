## Why

原始 `docs/ui.md` 设计文档描述了大量UI功能，但随着前端实现推进，已实现与未实现功能的差异日益模糊。需要重新组织该文档，明确指出哪些交互流程已实现、哪些未实现、哪些缺失，以便后续开发和需求管理。

## What Changes

1. **重新组织 `docs/ui.md`**：将原始设计文档转换为"设计vs实现"对比表格
2. **新增状态标记**：每项功能标记为 `✅已实现` | `🔄部分实现` | `❌未实现` | `📋规划中`
3. **补充实现证据**：已实现功能标注对应的源文件和组件路径
4. **补充缺失项**：设计中遗漏但需补充的功能项
5. **精简描述**：移除重复描述，保持1-2页简洁篇幅

### Modified Capabilities
