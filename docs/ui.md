CC Assistant UI 布局

1.会话界面
标题栏：
左1-会话名称（会话tab1/会话tab2/。。。，内嵌：重命名事件，新建会话名称为字母+0，用户发起对话后会变成用户的prompt，参考现在的大模型web对话的处理方式）；
右1-新建会话按钮；
右2-历史会话按钮；设置按钮
右3-流式输出开关；
消息区域:
用户消息（内嵌：复制按钮）；
ai消息（内嵌：复制按钮，在消息头和消息尾部都有）
消息行为：流式输出，代码块diff（新增、修改）
消息类型：markdown文本、代码块、超链接等
对话区：
左1-附件栏：接收图片，文件
右1-上下文占用比；
输入框；
工具栏-左：供应商悬浮列表(切换模型供应商)，对话模式悬浮列表（auto/plan/agent），思考模式开关，智能体悬浮列表，
工具栏-右：发送按钮（状态：发送中，可主动打断），提示词强化


2.历史会话界面
搜索栏：名称模糊搜索
会话列表：会话发起时间，用户提问次数
点击会话会将会话加载至当前会话tab；
会话列表控件：重命名会话，收藏按钮，导出，删除按钮

3.收藏会话界面（内嵌在历史会话界面的右下角，有一个大大的触发界面跳转按钮）
搜索栏：名称模糊搜索
会话列表：会话发起时间，用户提问次数
点击会话会将会话加载至当前会话tab；
会话列表控件：导出，删除按钮

4.设置界面（左右布局，左边菜单栏，右边配置界面）
4.1 供应商管理界面
新增供应商按钮；
供应商列表：供应商名称，状态，修改，导出json，删除
4.1.1 新增/修改 供应商界面
【1】标题栏： 新增供应商/编辑供应商：{供应商名称}
三方-快捷配置：Claude/GLM/Minimax 选中一个，自动填充下面除了apiKey以外的信息
【2】供应商名称
【3】url
【4】apiKey
【5】模型预设：默认，opus，max
【6】Json配置化：~/.claude/settings.json的格式化，会跟随上面填充的信息自动补全，同理也可以在这里一键填入json反映射到上面的几个控件
